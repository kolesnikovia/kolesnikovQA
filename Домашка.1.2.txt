DNS это распределительная система с информацией о доменах. Например пришел в гостиницу на ресепшн, попросил номер люкс, админ сходил за ключами и принес их вам.
Также DNS предоставляет юзеру информацию (айпи) сайта , на который тот хочет попасть.

Разница между http и https в том что второй более защищенный.
Для https нужен SSL сертификат который подписан центром сертификации.
http работает через порт 80, http - через 443.
Пишутся в URL по разному  http:// и https://.
https требует проверку домена а иногда и проверку юридического документа, а http - нет.
https шифрует данные перед отправкой, а http - нет.

HTTP 0.9 имеет основы синтаксиса и семантики протокола HTTP. был только метод GET
в HTTP 1.0 добавились методы HEAD и POST. Запросы стали начинаться с версии протокола, ответ начинался с кода статуса ответа.
в HTTP 1.1 появилось постоянное соединение, то есть можно было отправить несколько запросов и получить несколько ответов в том порядке в которым были заданы запросы
в рамках одного TCP соединения. Появилось КЭШирование.  Появилось согласование языка, кодировки или типа данных между клиентом и сервером
Добавились методы PUT, DELETE, OPTIONS, TRACE.
HTTP 2.0 стал быстрее. появились потоки (двунаправленность) в TCP. появилось много параллельных потоков в одном TCP.
Сервер стал самостоятельно иницииорвать передачу данных. Бинарная передача данных заместо текстовой
в HTTP 3.0 используется новый протокол QUIC. HTTP 3.0 нужен чтобы работало на множестве различных устройств. лучшее шифрование с помощью TLS 1.3 как в HTTPS.
